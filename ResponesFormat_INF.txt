指令发送逻辑：
1. 响应以':'结尾：
	-> 说明当前泵空闲，可以发送下一条指令
	-> receive_status = True
2. 响应以'>'结尾：
	-> 说明泵当前的状态为注入INF，此时以间隔0.1s的时间向其发送irate指令（绘制当前INF的流量曲线）
	   和crate指令，绘制the rate the motor running at
	-> receive_status = True
3. 响应以'<'结尾：
	-> 说明泵当前的状态为WD，同理，发送wrate和crate
	-> receive_status = True
4. 响应以'*'结尾：
	-> 说明泵当前被阻塞，停止向其发送指令
	-> receive_status = False
5. 响应以'T*'结尾：
	-> 泵当前设定的target已经达到，停止向其发送指令
	-> receive_status = False
